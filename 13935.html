    Java 程序是半编译、半解释的。Java 源程序经编译器编译后产生字节码（bytecode），Java 解释器解释执行字节码 。

    优点:

    • 兼具编译执行的效率优势和解释执行的灵活性；

    • 提高了应用程序的可移植性：源程序、中间代码

    Java 可移植性源于两方面：

    • Java 的半编译、半解释的特征

    • Java 采用标准的独立于硬件平台的数据类型

    • a = 00001111，则 a<<2 = 00111100

    • 在不产生溢出的情况下，左移一位相当于乘 2

      128 << 1 returns 128 * 2^1 = 256

      16 << 2 returns 16 * 2^2 = 64

    • 运算符 >> 执行一个右移位（带符号），左边按符号位补 0 或 1，例如：

      a = 00110111，则 a>>2=00001101

      b = 11010011，则 b>>2 = 11110100

      右移一位相当于除 2 取商

      例如： 128 >> 1 returns 128/ 2^1 = 64

      256 >> 4 returns 256/ 2^4 = 16

      -256 >> 4 returns -256/ 2^4 = -16

    • >>> : 同样是执行一个右移位，只是它执行的是不带符号的移位。也就是说对以补码表示的二进制数操作时，在带符号的右移中，右移后左边留下的空位中添入的是原数的符号位（正数为 0，负数为 1）；在不带符号的右移中，右移后左边留下的空位中添入的一律是 0

      a = 00110111， 则 a>>>2 = 00001101

      b = 11010011， 则 b>>>2=00110100

    • 一个类如果实现了 Serializable 接口，则这个类创建的对象就是可序列化的对象。声明为 transient 或 static 的变量不能被序列化。

    • 序列化的过程就是将对象（状态）写入字节流和从字节流中读取对象（状态）并在内存中重构对象。将对象状态转换成字节流之后，可以用 java.io 包中的各种字节流类将其保存到文件中、通过管道传递到另一线程中或通过网络连接将对象数据发送到另一主机。

    • 序列化分为两部分：序列化和反序列化。

      序列化是将数据分解成字节流，以便存储在文件中或在网络上传输。

      反序列化是打开、输入字节流并在内存中重构对象。

      对象序列化不仅要将基本数据类型转换成字节表示，有时还要恢复数据。恢复数据要求有恢复数据的对象实例。

    • ObjectOutputStream 使用 writeObject(Object obj)方法将一个内存中的对象 obj 序列化后写到一个文件中（序列化过程）。

      ObjectInputStream 使用 readObject()方法读取对象的序列化数据，并在内存中重构一个对象（反序列化过程）。

简答题的要点

  1. 类之间的 is-a 关系和 has-a 关系的含义。

    Student 和 Person 类对现实世界中，Student 具有 Person 的全部行为这种关系进行了建模。我们称这种关系为“是一个”关系。只有在一个类和所提议的派生类之间存在“是一个”关系时，才应该使用继承。（阅读新教材：8.1.1 派生类）

    如果两个提议的类之间不存在“是一个”关系，那么就不要用继承从一个类派生出另一个类。我们应该考虑在一个类的内部将另一个类的对象定义为实例变量。这种关系被称为“有一个”关系。

    另一种方式称为“有一个”关系。例如，我们有一个记录日期的 Date 类，通过在 Student 类中添加 Date 类型的实例变量，就可以在 Student 类中添加录取日期。在这种情况下，我们称一个 Student“有一个”Date 。

  3. 异常处理的运行机制。

    处理异常的结构由 try,catch,finally 构成，Java 运行出现非正常情况或错误时就会在 try 的块中抛出一个异常对象，在 catch 中有对应的函数，函数的参数是异常类类型，来接收从 try 中抛出的异常对象并执行 catch 中的操作；finally 部分无论是否出现异常最后都会执行。

    异常被捕获: finally 代码正常执行并且 finally 后面正常的代码还是可以继续执行

    异常未被捕获: finally 代码正常执行，并将异常向上一层调用方抛出，但是 finally 后面正常的代码不再执行

  4. 事件处理机制的原理。

    事件监听器先注册事件源，当用户发生例如点击鼠标的动作时会触发事件源生成事件的对象，事件会触发监听器，事件的对象作为监听器这个类中的事件处理的方法的函数参数传入，然后接受参数的方法做出响应。
