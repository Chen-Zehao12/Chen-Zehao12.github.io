第 3 章 语言和文法的形式定义

发表于2022-01-09|更新于2022-01-21|编译原理
|字数总计:864|阅读时长:2分钟

1. 文法、语言？

  • 语言的有穷表示
    • 生成方式（文法）
      • 语言中的每个句子可以用 严格定义的规则 来构造
    • 识别方式（自动机）

2. 文法的分类是怎样的？它们之间有何关系？

  • 文法
    • 四个基本要素
      • 终结符集合VTV_TVT​
      • 非终结符集合VNV_NVN​
      • 开始符号SSS
      • 规则或产生式的集合PPP
    • 注意
      • 规则右部的运算符是选择、并置、括号
      • 没有重复运算符，用递归实现重复
      • 右部可以使用ϵ\epsilonϵ
  • 通过对 产生式 施加不同的限制，把文法及其对应的语言分成四种类型，即0 型、1 型、2 型和 3 型
  • 0 型文法
    • 规则左侧至少包含一个非终结符
    • 能够用 0 型文法定义的语言称为 0 型语言或递归可枚举语言
  • 1 型文法
    • 左侧的长度≤右侧的长度，仅 S→ϵS→\epsilonS→ϵ 除外
    • 要求 SSS 不得出现在任何产生式的右部
    • 也称为上下文有关文法
    • 能够用 1 型文法定义的语言称为 1 型语言或上下文有关语言
  • 2 型文法
    • 左侧都是非终结符号
    • 能够用 2 型文法定义的语言称为 2 型语言或上下文无关语言
  • 3 型文法
    • 右线性规则或左线性规则
    • 能够用 3 型文法定义的语言称为 3 型语言或正规语言
    • 3 型文法也称为正则文法（或正则文法）

3. 推导、规约、语法树、文法的二义性？

  • 推导：从左到右

  • 规约：从右到左

  • 规范推导 = 最右推导

  • 规范规约 = 最左规约

    • 凡是句子，必定存在规范规约

  • 分析树可以反映推导的全过程，信息齐全，但过于复杂，耗费空间

  • 因此需要压缩，只存储有用信息——语法树，但无法还原记号序列，比分析树效率更高

  • 文法的二义性

    • 对于一个文法的同一个句子而言，若存在两个不同的语法树与之对应，则称该文法是二义性的

  • 一个 CFG 是否为二义的问题是不可判定的，即不存在解决该问题的算法

  • 消除二义性

    • 没有通用的办法可以消除文法的二义性，但在实践中，对于常用的文法，可以找到特定的消除歧义性的办法

6. 文法的构建问题

  • 反映运算符号间优先关系的规则
  • 在树型中，离根越近，则优先级越低
  • 左递归规则可实现左结合，而右递归规则可实现右结合
  • 文法中不含有有害规则和多余规则
  • 左公因子
    • 提取左公因子
    • 改写为两个规则
  • 消除回溯性
    • 求 first 集合
  • 直接左递归
    • 用 EBNF
    • 改写为右递归
